# taste-design Audit — e-motion Warisan Edition

> Generated by the `/taste-design` skill as an **audit lens** (option A — no files changed).
> The `taste-design` skill encodes a *minimalist / gallery-airy / anti-ornament* design
> philosophy. The Warisan brand is deliberately *ceremonial / ornamental / heritage-rich*.
> Many "violations" below are **intentional brand choices**, not slop. Each row is tagged:
>
> - **KEEP** — the Warisan brand deliberately overrides this taste-design rule; the choice is correct for a Malay royal-court heritage product.
> - **FIX** — genuine AI-slop that taste-design correctly catches; worth cleaning up.
> - **JUDGE** — depends on intent; user decides.

---

## 1. Typography

| taste-design rule | Warisan reality | Verdict | Notes |
|---|---|---|---|
| `Inter` BANNED for premium | Inter used for eyebrows, buttons, tier badges, nav, form inputs | **KEEP** | Inter is the *utilitarian* layer against two display serifs — a deliberate 3-tier contrast. Swapping to Geist/Satoshi would not change the perceived quality; the serifs carry the premium feel. Low-value change. |
| Generic serifs BANNED (Playfair/Garamond/Georgia) | Playfair Display + Cormorant Garamond are the entire display system | **KEEP** | This is the single most consequential disagreement. Playfair Italic + Cormorant *is* the Warisan identity — ceremonial, wedding-invitation, jawi-adjacent. taste-design's allowed serifs (Fraunces, Editorial New) read modern-editorial, not ceremonial-Malay. Overriding this rule is correct. |
| Mono for numbers when density > 7 | No mono font loaded | **JUDGE** | Warisan density is ~4 (airy ceremonial), so the rule doesn't trigger. Fine as-is. |

## 2. Color

| taste-design rule | Warisan reality | Verdict | Notes |
|---|---|---|---|
| Max 1 accent, saturation < 80% | Gold (#c9a24e ~52% sat) is the single accent — maroon/emerald are *surface* tones not accents | **KEEP** | Compliant in spirit. Gold is the lone accent; the indigo/maroon/emerald are mood-surfaces. |
| No pure black `#000000` | `--indigo-deep` #0d1228 is the floor — never pure black | **KEEP** | Already compliant. |
| No neon / outer-glow shadows | Gold "halo glow" on card hover (`0 0 40px rgba(201,162,78,0.18)`) | **JUDGE** | This is a *candlelight* glow, low-opacity, warm — not neon. Defensible as atmosphere. Could soften further if it reads as glow-y. |
| No oversaturated accents | Gold is calibrated, sub-60% saturation | **KEEP** | Compliant. |
| No excessive gradient text on large headers | Card titles + some headings use gold-gradient `-webkit-background-clip:text` | **FIX** | This one lands. Gradient text-fill on headings is a mild AI tell. Recommend: solid `var(--gold-light)` on headings; reserve the gradient strictly for the small tier-pill badges and CTAs. |

## 4. Content integrity

| taste-design rule | Warisan reality | Verdict | Notes |

## 5. Motion

| taste-design rule | Warisan reality | Verdict | Notes |
|---|---|---|---|
| Spring physics, no linear easing | Hover transitions use `cubic-bezier(.19,1,.22,1)` (an ease-out, spring-ish) | **KEEP** | Close enough — the bezier reads weighty. Not linear. Fine. |
| Animate only `transform`/`opacity` | Particles animate `transform: translateY` ✓; marquee uses `transform` ✓; but `.hero-scroll-line::after` animates `top` | **FIX** | The legacy scroll-cue line animates `top` — a layout-thrashing property. Moot if the scroll cue is deleted per §3. |
| Perpetual micro-loops on active components | Gold dust drifts, marquee loops, songket weave pans | **KEEP** | Strongly compliant — this is a Warisan strength. |
| Staggered cascade reveals | `reveal-3d reveal-d1..d7` stagger system | **KEEP** | Compliant. |

## Recommended FIX set (the genuine slop — no brand conflict)

If you want a **selective cleanup** (taste-design option C) without abandoning the Warisan brand, apply only these:

1. **Kill the fabricated stats** — rework `StatsStrip` to real/non-numeric value props.
2. **Delete scroll cues** — remove `hero-asym-scroll` + the undang `cover` chevron.
3. **De-emoji** — replace `♥`/`❤`/`✦` with inline gold SVG or nothing.
4. **`100vh → 100dvh`** — global swap in `warisan.css` hero/section classes (iOS Safari fix).
5. **Heading gradient text** — solid `var(--gold-light)` on headings; keep gradient only on pills + CTAs.
6. **`top` animation** — removed for free once the scroll cue is deleted.

Everything else (Playfair/Cormorant, Inter, ornaments, glass, overlapping vignette, gold halo) is **intentional Warisan heritage identity** and should NOT be "fixed."
